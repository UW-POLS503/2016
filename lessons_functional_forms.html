Intro

I use as a source of inspiration in this lesson Chris Adolph’s POLS-503 notes.

In a linear model, any unit change of a given covariate has always the same effect on the outcome variable. If the coefficient for that covariate is for example 0.54, it doesn’t matter if the unit change is from 1 to 2 or from 100,001 to 100,002. The expected effect on the outcome variable is always the same: 0.54 increase in the units of the dependent variable.

However, theoretically we may sometimes think that the way a covariate affects the response variable is not linear. For example, is the relationship between income and life expectancy linear? Or Acemoglu and Robinson (2006) argue that the relationship between inequality and democratization follows a U-shape (we’ll talk about these two examples in more detail next). How do we deal with this non-linear relationships? Can we still use a linear models to estimate them?

\(y = \beta_{0} + \beta_{1}X_{1} + \beta_{2}X_{2}... + \epsilon\)

Logrithmic Transformations

Load the gapminder dataset from the gapminder package. Imagine that lifeExp is simply a linear function of gdpPercap.

\(lifeExp = \beta_{0} + \beta_{1}gdpPercap + \epsilon\)

Plot the outcome variable lifeExp against the response variable gdpPercap. Include a linear regression line (geom_smoth()) to the plot (with no confidence interal: se = FALSE)

ggplot(gapminder, aes(y = lifeExp, x = gdpPercap)) +
  geom_point() +
  geom_smooth(method = "lm", se = FALSE)

This graphs clearly shows that in the beginning, a unit increase on gdpPercap matters a lot in explaining lifeExp. However, at some point, extra units increase do not make much of a difference. We call this type of relation a Logarithmic Relation. How can we specify this logarithmic relation in a linear model?

If we believe that there is a logarithmic relation between the outcome variable and a covariate, this is the same as saying that there is a linear relation between the outcome variable and the \(log\) of the covariate. Thus, we just need to transform the covariate of interest by calculating its \(log\) and then add the resulting variable to the linear model.

To show this, include a new variable to the gapminder dataset: the log of gdpPercap.

gapminder <- mutate(gapminder, log_gdpPercap = log(gdpPercap))

Compare the distribution of gdpPercap and log_gdpPercap by looking at their quantiles().

data.frame(qGdp = quantile(gapminder$gdpPercap),
           qLogGdp = quantile(gapminder$log_gdpPercap))
##             qGdp   qLogGdp
## 0%      241.1659  5.485485
## 25%    1202.0603  7.091792
## 50%    3531.8470  8.169576
## 75%    9325.4623  9.140504
## 100% 113523.1329 11.639762

Plot lifeExp against this new variable. Include again a linear regression line to the plot. Observe how there is a linear relation between the outcome and the log of the covariate. Observe also that log_gdpPercap is not expressed on the original units ($ per capita) anymore but in log $ per capita.

ggplot(gapminder, aes(x = log_gdpPercap, y = lifeExp)) +
  geom_point() +
  geom_smooth(method = "lm")

The model specification describing the two regressions are.

mod1a <- lm(lifeExp ~ gdpPercap, data = gapminder)
mod1b <- lm(lifeExp ~ log(gdpPercap), data = gapminder)

Let’s see how well each model does in predicting the actual lifeExp values.

gapminder_augm1 <- augment(mod1a, gapminder)
gapminder_augm2 <- augment(mod1b, gapminder)  
ggplot(gapminder_augm1, aes(x = lifeExp, y = .fitted)) + 
  geom_point()

ggplot(gapminder_augm2, aes(x = lifeExp, y = .fitted)) +
  geom_point()

Challenge

How would you create a dataset that contains the actual and predicted values of both models? Having such dataset would facilitate creating a side-by-side plot using the facet_wrap of ggplot.

Solution 

predictions <- NULL
models_list <- list(mod1a, mod1b)
for (i in 1:length(models_list)) {
  m <- models_list[[i]]
  new_data <- data.frame(actual = m$model$lifeExp,
                         predicted = m$fitted.values,
                         model = i)
  predictions <- rbind(predictions, new_data)
}
ggplot(predictions, aes(x = actual, y = predicted)) +
  geom_point() +
  facet_wrap(~ model)

Challenge

  1. Look at the regression table for mod1b. How would you interpret the coefficients?
  2. Can you think of any covariate that you use in your own projects that has a logarithmic relation with the outcome variable you study? Describe that relationship.

Squared Terms

As we discussed in the beginning, sometimes there is a squared relation between a covariate and our dependent variable of interest (e.g. Kuznets curve).

Load the turnout dataset from the uwpols501 package. Imagine that income is simply a linear function of age.

Plot the outcome variable age against the response variable income. Include a conditional mean line (geom_smoth()) and a linear regression line. Do not plot confidence intervals for these lines, and choose a different color for each of the two.

ggplot(turnout, aes(y = income, x = age)) +
  geom_point() +
  geom_smooth(method = "lm", se = FALSE) +
  geom_smooth(se = FALSE, color = "red")

Challenge

Which of the two lines do you think describes better the data and makes the most theoretical sense?

Probably the one that describes a more squared (U-shape) relation:

  • People do not make much money when they start working (age between 18 and 25).
  • People’s income increases as age and working experience also increases (from 25 to around 65).
  • When people retire, their income decreases. Not everybody retires at the same age. However, beyond a certain point (e.g. 65), as people get older, the likelihood of their income decreasing increases.

Plot the squared otucome variable against income.

ggplot(turnout, aes(y = income, x = age^2)) +
  geom_point() +
  geom_smooth(method = "lm", se = FALSE) +
  geom_smooth(se = FALSE, color = "red")

Observe that in this second plot, as age increases, we’re are making the 1-year increases larger. This means that in this new variable we’re giving more weight to larger values of age. If we add in a model both the non-squared (age) and the squared term (age^2), the coefficient for age will indicate the effect of 1-year increase after controlling for the quadratic effect of age. In other words, after controlling for the fact that older individuals behave differently.

On the other hand, a positive coefficient for age^2 will mean that an increase of age has even a bigger effect on income when people gets older. A negative effect will mean the opposite, than when people gets older age has a negative effect on income (or even more negative if the effect of the non-squared term was alread negative).

Run the following regression:

mod_age <- lm(income ~ age, data = turnout)
sq_age <- turnout$age^2
mod_age_sq <- lm(income ~ age + sq_age, data = turnout)

Compare the coefficients and standard errors of the covariates

Compared to logarithimc terms, the coefficients for squared terms are a lot harder to interpret. I suggest you use simulations and predictions to do so.

For example, create a vector with 100 values of age that range from the minimum to the maximum value of age in the turnout dataset. Then use the vector to create a new dataset with the same covariates in mod2: age and sq_age.
